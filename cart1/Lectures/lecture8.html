A Note on Generalization

Generalization comes in to play in three ways with Choropleth Maps.

  • Standardization of data based on area of enumeration unit
  • Categorization of data values for representation
  • Visual generalization of map representation

Classed vs. Unclassed Maps

  • Classed- data is represented in classes
  • Unclassed- data is not binned so the value is s

Data Normalization

RULE: You may never use choropleths to represent raw count data.

  • Counts or totals and measures are summary statistics for the unit and are only true when they represent the unit as a whole
  • If you divide that whole, then the value changes (for example, moving from census tract to census block the count changes)
  • Density, Percentages, etc. are not dependent upon the unit, as they have been normalized with the same denominator

Data Normalization

For an intersting article on this topic see Buckley (2013)

  • Ratio: relationship of one item type to another (Number of goats to Number of Farms)
  • Percentage: number of items divided by total number of items and multiplied by 100 (10% of the population)
  • Proportion: generated by dividing the number of items in a unit by the total number of items
  • Density: relationship of one observation to unit area (People per Square Mile)

Color

Color Scales are used to map data values to color representation. Three main color scales are used for single-theme choropleth maps.

  • Sequential - variation in color value from light (lower values) to dark (high values)
  • Diverging - opposing color variations from a central/neutral color
  • Qualitative - variation in color hue to represent individual classes

Bivariate Color Schemes

Bivariate choropleths combine color scales for two variables
